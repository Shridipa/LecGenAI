01 Signal Processing & Samples

The raw material — every sound is just a 1D array of numbers

Before a single line of transcription can run, you need to understand what an audio stream actually is at a mathematical level. Every second of audio captured is a 1-dimensional NumPy array representing sound pressure levels over time. There are no "voices" in memory — only floating-point integers sampled at high frequencies.

# Waveform properties for LecGen Optimizers audio.shape = (16000, 1) │ │ └─ Mono channel (LecGen standard) └──── 16k samples per second (16kHz) ┌─────────────────────────────────────────────────────┐ │ Sample 0 → [ -0.0023 ] │ │ Sample 1 → [ -0.0021 ] │ │ ... │ │ Sample 15999 → [ 0.0452 ] │ └─────────────────────────────────────────────────────┘ Each float → -1.0 to 1.0 ← PCM Float Standard Bitrate Encoding → 64k (Compressed for sub-1m speed)
The Sample Rate Trap

Mismatched sample rates lead to "aliasing" distortion. While video is captured at 48kHz, neural models like Whisper expect exactly 16kHz for accuracy. LecGen uses FFmpeg to resample input streams instantly using cv2.decode_b64_frame logic adapted for audio buffers. Always ensure mono-conversion before processing.

Key Preprocessing Operations

Operation Implementation Engine Rationale
Resample ffmpeg -ar 16000 aligns with HF model input requirements
Bitrate Cap 64k LAME Encoder Halves payload size; reduces latency to <2s
VAD Filter model.transcribe(vad_filter=True) Skips "dead sound" to avoid neural hallucinations

02 Neural Transcription — Whisper

Converting raw samples to semantic meshes — text is just a high-level representation

Once the signal is clean, LecGen's primary engine (Whisper Base) initiates the transcription mesh. Unlike simple word-matchers, Whisper uses an Encoder-Decoder Transformer architecture that doesn't just "hear" words—it understands context, allowing it to differentiate between "there" and "their" or "neural" and "new real" based on the surrounding sentence structure.

03 Language Synthesis & Summaries

Mapping precise anatomical points onto the textual content

Once you have the text mesh, LecGen's summarization engine (DistilBART-6-6) maps out specific "knowledge keypoints" — topic headers, bullet hierarchies, and core concepts. This is categorically more detailed than a simple summary; it is a structural reconstruction of the lecture's internal logic.

Topic Grouping

Clusters related sentences into coherent study modules based on semantic similarity.

Logic Pruning

Removes filler words, stuttering, and non-essential anecdotes to reveal the "Mastery Path".

Hierarchical Naming

Extracts Noun Phrases to automatically name sections (e.g., "Matrix Calculus" vs "That math thing").

04-05 From Embeddings to Knowledge Nodes

How deep networks compress a lecture into a meaningful study fingerprint

A raw transcript is a high-dimensional object. LecGen acts as a learned compression function: it maps enormously noisy text down to compact Knowledge Embeddings (via T5-Small). These embeddings are then evaluated against Distance Heuristics to ensure accuracy.

The Knowledge Heuristics

  • Evaluative Scoring (40%)

    We run RoBERTa-based SQuAD models to cross-verify every generated quiz answer against the original transcript samples.

  • Speed Weighting (20%)

    Parallelized generation streams (Notes + Quiz + Flashcards) fire simultaneously via a dedicated ThreadPoolExecutor.
